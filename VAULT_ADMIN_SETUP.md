# Vault Admin Setup

1. Create a `.env` file in the project root using `.env.example`.
2. Set a strong `ADMIN_PASSPHRASE` and `SESSION_SECRET`.
3. Add your Supabase project URL and service role key if you want production data stored in Supabase.
4. Start the site and API with:

```powershell
python server.py
```

5. Open `http://localhost:8000/vault-admin.html`.
6. Enter the passphrase to unlock the admin actions.

## Supabase setup

Create these tables in the Supabase SQL editor:

```sql
create table if not exists public.rates (
  id bigint generated by default as identity primary key,
  rate_22k_1g numeric not null,
  rate_22k_8g numeric not null,
  rate_18k_1g numeric not null,
  silver_1g numeric not null,
  market_status text not null,
  created_at timestamptz not null
);

create table if not exists public.uploads (
  id bigint generated by default as identity primary key,
  title text not null,
  tag text not null,
  file_path text not null,
  original_name text not null,
  created_at timestamptz not null
);
```

Create a public Storage bucket named `vault-uploads`.

Add these values to `.env`:

```env
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
SUPABASE_STORAGE_BUCKET=vault-uploads
```

The service role key stays on the backend only. Do not put it in any HTML or frontend JavaScript file.

## What the backend does

- `POST /api/admin/login`: checks the passphrase and returns a signed admin token.
- `POST /api/update-rate`: saves the latest gold and silver rates into Supabase when configured, otherwise SQLite.
- `POST /api/upload-image`: uploads the image to Supabase Storage when configured, otherwise `uploads/`, and records metadata.
- `GET /api/rates`: returns the latest published rates.
- `GET /api/uploads`: returns recent uploaded collection items.

## What to use in production

- Host this behind HTTPS.
- Keep `.env` on the server only.
- Do not expose `ADMIN_PASSPHRASE` or `SESSION_SECRET` in frontend files.
- If using local SQLite, back up `data/vault.db` and the `uploads/` folder.
- If you deploy to a VPS, run `server.py` behind Nginx or Caddy.
